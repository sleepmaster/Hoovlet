Here you can see list of currently cached items by this DryDrop site.

DryDrop site acts as a cache for your GitHub content. When someone requests resource at given URL, DryDrop looks into the cache and serve item directly whenever possible. In a case the cache entry is missing, DryDrop tries to fetch the item from GitHub and store it in cache for later requests.

What about updating? In case you have correctly setup GitHub post commit hook for your repo. When you push updated content to your GitHub repository, GitHub will ping DryDrop and DryDrop then invalidates all changed items by that push. This way next time when someone requests updated item cache miss will happen and DryDrop downloads fresh content from GitHub.
